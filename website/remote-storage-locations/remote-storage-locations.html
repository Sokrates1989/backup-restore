üß≠ SFTP Setup (Ubuntu 22.04 example)

Server-side checklist:

  • Install SSH server: sudo apt update && sudo apt install -y openssh-server
  • Ensure port 22 is reachable: sudo ufw allow 22/tcp (or open your firewall/security group)
  • Create a user: sudo adduser backup
  • Create remote folder: sudo mkdir -p /backups
  • Grant access: sudo chown -R backup:backup /backups

For production, consider SSH key auth and disabling passwords.

üîê SSH key auth + disable passwords (recommended)

1) Create a key pair (client machine)

ssh-keygen -t ed25519 -a 100 -f ~/.ssh/backup_id_ed25519

2) Install the public key for the SFTP user

Either via:

ssh-copy-id -i ~/.ssh/backup_id_ed25519.pub backup@your-server

Or manually:

  • Create folder: sudo -u backup mkdir -p /home/backup/.ssh
  • Append key: sudo sh -c 'cat backup_id_ed25519.pub >> /home/backup/.ssh/authorized_keys'
  • Fix perms: sudo chown -R backup:backup /home/backup/.ssh && sudo chmod 700 /home/backup/.ssh && sudo chmod 600 /home/backup/.ssh/authorized_keys

3) Enable key auth in SSH and (optionally) disable passwords

Edit: sudo nano /etc/ssh/sshd_config

  • Ensure: PubkeyAuthentication yes
  • Optional hardening: PasswordAuthentication no
  • Optional hardening: KbdInteractiveAuthentication no

4) Restart SSH

sudo systemctl restart ssh

5) Test login before disabling passwords

ssh -i ~/.ssh/backup_id_ed25519 backup@your-server

If you disable passwords, keep an active root/console access to avoid locking yourself out.

If provided, the connection test uses the private key instead of password.

Steps:

  • Create a Google Cloud project and enable Google Drive API
  • Create a Service Account and generate a JSON key
  • Create a folder in Google Drive and share it with the service account email
  • Copy the folder id from the URL and paste it below
  • Paste the JSON key below (stored encrypted in the database)

Folder URL example:
https://drive.google.com/drive/folders/<FOLDER_ID>?usp=sharing
Only the <FOLDER_ID> part is needed.

This is a server-managed integration. OAuth user-login flow is not implemented yet.

You can paste the JSON or upload the downloaded .json key file.
